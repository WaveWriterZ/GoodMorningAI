Security Policy

Security Philosophy

GoodMorningAI and the Mission Systems ecosystem are committed to building secure, transparent, and trustworthy software.

Security is considered an ongoing responsibility shared by maintainers and contributors.

---

Reporting a Vulnerability

If you discover a security issue, please do not create a public issue if it could put users or the project at risk.

Instead, report the issue privately to the project maintainers with:

- A description of the issue.
- Steps to reproduce it.
- The potential impact.
- Any suggested mitigation or fix.

Reports should include enough detail to help us investigate and resolve the issue efficiently.

---

Response Process

When a valid security issue is reported, the project aims to:

1. Acknowledge receipt of the report.
2. Investigate the issue.
3. Develop and test a fix.
4. Release the fix.
5. Document the resolution where appropriate.

Response times may vary depending on project resources and the severity of the issue.

---

Supported Versions

During the early stages of development, only the latest version of the project is considered supported.

As the project matures, supported versions and maintenance timelines will be documented here.

---

Responsible Disclosure

We appreciate responsible disclosure practices that help protect users while allowing maintainers time to investigate and address reported issues.

---

Our Commitment

We are committed to continuously improving the security, reliability, and resilience of GoodMorningAI and the Mission Systems ecosystem through responsible engineering and community collaboration.
